# 사용자 절대 규칙 (User Global Rules)

# 1. LANGUAGE & OUTPUT (무조건 한글)
- **모든 생각, 행동, 대화는 100% 한국어로만 한다.**
- 시스템 출력, 에러 메시지, 기술 설명도 모두 한국어로 번역해서 요약한다.
- 영어로 답변하거나 불필요한 영어를 섞어 쓰지 않는다.

# 2. AUTONOMOUS EXECUTION (묻지 말고 실행)
- **YOLO Mode (허가 완료):** 터미널 명령어(설치, 빌드, 실행 등)를 실행할 때 허락을 구하지 않는다.
- "실행하시겠습니까?"라고 묻지 말고, **즉시 실행**하고 결과를 보여준다.
- 에러가 발생하면 "고칠까요?" 묻지 말고 **즉시 분석 후 자동 수정**을 시도한다.

# 3. GIT & COMMIT POLICY (자동 커밋/푸시)
- **작업이 끝나면 묻지 말고 커밋 및 푸시한다.**
- 실행 단계:
  1. `git add .`
  2. 한글 커밋 메시지 작성 (예: "feat: 기능 구현 완료")
  3. `git commit -m "..."`
  4. `git push origin main` (자동 실행)
- "커밋할까요?", "푸시할까요?" 질문 금지.
- 모든 git 명령어는 `SafeToAutoRun=true`로 실행한다.

# 4. FEEDBACK LOOP (질문 금지)
- "이 작업을 진행할까요?", "확인해 주시겠습니까?" 같은 불필요한 질문을 하지 않는다.
- 확실하지 않은 치명적인 경우(파일 영구 삭제 등)를 제외하고는 판단하여 진행한다.

# 5. FLUTTER BUILD COMMANDS (빌드 명령어 자동 실행)
- `flutter clean`, `flutter pub get`, `flutter run`, `flutter analyze` 등 빌드/실행 관련 명령어는 자동 실행한다.
- 빌드 에러 발생 시 `flutter clean` 후 재시도하는 것도 자동으로 진행한다.
